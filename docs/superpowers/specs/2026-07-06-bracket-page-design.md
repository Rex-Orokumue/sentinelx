# Bracket Page — Design

**Roadmap task:** v1.0 #4 · **Route:** `/tournaments/[slug]/bracket`
**Date:** 2026-07-06

## Purpose

A public, read-only bracket page showing a tournament's group stage (standings +
fixtures) and knockout tree. It reflects **admin-confirmed** data on each load —
brackets are generated by an admin after registration closes (task #9); this page
only displays whatever `groups` / `group_memberships` / `matches` rows exist.

## Constraints already in place (verified, no migration needed)

- `groups`, `group_memberships`, `matches`, `profiles` all have `public_read`
  (`USING (true)`) RLS — the anon server client reads everything this page needs.
- `matches.player_a_id` / `player_b_id` are **NOT NULL** → knockout matches only
  exist once both players are known (no TBD placeholder rows).
- `group_memberships` stores `points, wins, draws, losses, goals_for, goals_against`
  → standings are a **sorted read**, not a computation. Played = W+D+L, GD = GF−GA.
- `matches.round` ∈ {`group`,`round_of_32`,`round_of_16`,`quarter_final`,`semi_final`,`final`}.
- `matches.status` ∈ {`scheduled`,`live`,`completed`,`disputed`,`cancelled`}.

## Architecture

Server Component page fetches all data; presentational children render it. The
**only** client boundary is `GroupStage` (tabs + collapse). No writes, **no realtime**
— the page reflects admin-confirmed DB state on each load (realtime is out of scope
for v1.0).

## Data fetch (one page load, in the page Server Component)

- Tournament by slug — `notFound()` if missing or `status = 'draft'` (reuse the
  detail page's `getTournament` pattern).
- `groups` for the tournament, ordered by `name`.
- `group_memberships` for those groups, join `profiles(id, username, display_name)`.
- `matches` for the tournament, join profiles for both players. Split in-code:
  - group matches (`group_id != null`)
  - knockout matches (`round != 'group'`)

## Page structure (stacked, sections conditional)

1. **Header** — back-link to `/tournaments/[slug]`, "Bracket" title, and a **champion
   banner** when the final is completed (`getChampion` returns non-null).
2. **Group Stage** (only if `groups.length > 0`) — client `GroupStage` with two tabs:
   - **Table** (default): each group's `StandingsTable` — rank, player, P/W/D/L/GD/Pts;
     top-`advancingCount` rows highlighted (✅ advancing).
   - **Fixtures**: **global** (all groups combined), each card tagged with its group,
     split into **🔴 Live** (top, pulsing) → **⏳ Upcoming** (chronological) →
     **🏁 Completed** (collapsed behind a "Show results" toggle). Global — not
     per-group — because this is a time-sorted "what's on now" surface.
3. **Knockout** (only if knockout matches exist) — `KnockoutBracket`: stacked round
   sections in canonical order (Round of 32 → Final), each a list of `MatchCard`s.
4. **Empty state** — no groups and no knockout matches → "Bracket not published yet.
   It'll appear here once registration closes and the admin sets it up."

## Components

- `components/bracket/GroupStage.tsx` — **client**; Table/Fixtures tabs; Fixtures
  Live/Upcoming/Completed grouping + the "Show results" collapse toggle.
- `components/bracket/StandingsTable.tsx` — presentational standings table for one group.
- `components/bracket/KnockoutBracket.tsx` — presentational stacked rounds.
- `components/bracket/MatchCard.tsx` — shared: player A vs B, score or "vs", status
  badge, optional group tag, links to `/matches/[id]` (Match Centre #5 — dead link
  until that page exists, consistent with the earlier detail→bracket link).

## Pure, testable logic (vitest, TDD)

### `lib/tournaments/standings.ts`

```ts
export interface StandingRow {
  playerId: string
  name: string
  played: number
  wins: number
  draws: number
  losses: number
  goalsFor: number
  goalsAgainst: number
  goalDiff: number
  points: number
  rank: number       // 1-based, after sort
  advancing: boolean // rank <= advancingCount
}

// Order: points desc, then goal difference desc, then goals-for desc.
// advancingCount defaults to 2 (top-2 advance) but is a parameter now so a
// future format (e.g. best third-place) needs no surgery.
export function sortStandings(
  memberships: MembershipInput[],
  advancingCount?: number, // default 2
): StandingRow[]
```

### `lib/tournaments/bracket.ts`

```ts
// Canonical knockout order — the single source of truth for round sorting,
// independent of DB insertion/return order.
const ROUND_ORDER = [
  'round_of_32',
  'round_of_16',
  'quarter_final',
  'semi_final',
  'final',
] as const

// A match is only "live" when an admin has set status = 'live' in the DB.
// With realtime out of scope for v1.0, the pulsing "Live" indicator reflects
// admin-confirmed state, not the actual real-time match state — it can lag.
export function splitFixturesByState(matches: MatchInput[]): {
  live: MatchInput[]
  upcoming: MatchInput[]   // status scheduled; sorted by scheduled_at (nulls last)
  completed: MatchInput[]  // status completed
}

// Only rounds that have matches, in ROUND_ORDER; each with a display label.
export function orderKnockoutRounds(matches: MatchInput[]): {
  round: string
  label: string
  matches: MatchInput[]
}[]

// The tournament champion: winner of the completed final, else null.
export function getChampion(matches: MatchInput[]): { id: string; name: string } | null
```

`disputed`/`cancelled` matches are excluded from live/upcoming/completed buckets
(they render, if at all, only within their knockout round list with a status badge).

## Error handling

- Missing/draft tournament → `notFound()`.
- No bracket data → empty state (not an error).
- Null scores render as "vs" (scheduled) / show the score once both present.
- Null `scheduled_at` sorts last in Upcoming.

## SEO

`generateMetadata()` → title `Bracket — {tournament title} | Sentinel X`, description,
OG tags. Result/bracket pages stay live permanently (SEO rule).

## Testing

Vitest units (pure functions, no mocking):

- `sortStandings` — ordering by points→GD→GF; tie-breaks; rank assignment; `advancing`
  flag honoring default 2 and a custom `advancingCount`; derived `played`/`goalDiff`.
- `splitFixturesByState` — bucketing by status; upcoming sorted by `scheduled_at` with
  nulls last; disputed/cancelled excluded.
- `orderKnockoutRounds` — rounds returned in `ROUND_ORDER` regardless of input order;
  empty rounds omitted; labels correct.
- `getChampion` — winner of completed final; null when final missing/incomplete;
  correct side (a vs b) by score.

Components and the page are verified by `tsc` + `lint` + `build` and manual review
(no data yet until admin tooling in #9; empty state is the default rendered state).
